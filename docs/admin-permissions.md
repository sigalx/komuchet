# Права Админки

Документ фиксирует текущую матрицу доступа административного интерфейса.

## Уровни Доступа

`ROLE_ADMIN` - глобальный администратор платформы. Признак выводится из `users.admin_granted_at is not null and users.admin_revoked_at is null`.

`WORKSPACE_ADMIN` - администратор текущего хозяйства. Право получает глобальный администратор или пользователь с активной ролью `admin` в `workspace_user_role_assignments`.

`WORKSPACE_ACCESS` - доступ к рабочей админке текущего хозяйства. Право получает глобальный администратор, администратор хозяйства или оператор хозяйства.

## Матрица

| Раздел или действие | Оператор хозяйства | Администратор хозяйства | Глобальный администратор |
| --- | --- | --- | --- |
| Рабочий стол `/admin` | Да | Да | Да |
| Рабочие справочники и операции хозяйства | Да | Да | Да |
| Пользователи: список и карточка | Да | Да | Да |
| Пользователи: создание, email, lifecycle, пароль | Нет | Да | Да |
| Пользователи: прямая связь `User` с `Subscriber` из карточки пользователя | Нет | Да | Да |
| Выдача и отзыв ролей хозяйства `admin`/`operator` | Нет | Да | Да |
| Подключение абонентского портала из карточки абонента | Да | Да | Да |
| Audit log текущего хозяйства | Нет | Да | Да |
| Глобальные audit-события и audit других хозяйств | Нет | Нет | Да |
| Реестр хозяйств `/admin/workspaces` | Нет | Нет | Да |
| Переключение текущего хозяйства | Только доступные хозяйства | Только доступные хозяйства | Все хозяйства |

## Правило Для Новых Разделов

По умолчанию новый рабочий раздел текущего хозяйства защищается `WORKSPACE_ACCESS`. Это подходит для участков, абонентов, счетчиков, показаний, оплат, начислений, расчетов, квитанций, доставок и платежных реквизитов.

`WORKSPACE_ADMIN` нужен для действий, которые управляют доступами, ролями, глобальной учетной записью пользователя или раскрывают audit log.

`ROLE_ADMIN` нужен только для платформенных действий вне одного хозяйства: управление реестром хозяйств и будущие глобальные настройки платформы.

Если действие финансово значимое, оно не обязательно требует `WORKSPACE_ADMIN`; оно должно быть доступно оператору хозяйства, но обязательно писать запись в `audit_logs`.
