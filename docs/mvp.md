# MVP

Первая версия системы должна закрыть ввод и просмотр показаний электроэнергии, а также дать администратору ручные инструменты для обслуживания абонентов.

Статус: вертикальный MVP в основном реализован. Документ остается границей первой версии и используется как контрольный список реализованного состава MVP.

## Входит В MVP

### Абонентский Интерфейс

- Вход в систему по email и паролю.
- Получение доступа к хозяйству после подключения учетной записи оператором или администратором.
- Просмотр доступных участков.
- Просмотр электросчетчика по участку.
- Передача нового показания.
- Просмотр истории переданных показаний.
- Просмотр потребления по расчетным периодам.
- Просмотр начислений с разбивкой по социальной норме и сверх нормы.
- Просмотр баланса или долга, если оплаты уже внесены администратором.
- Просмотр экрана "Баланс и операции" по участку.
- Просмотр сохраненных системных квитанций с PDF, если они уже сформированы.

### Административный Интерфейс

- Поиск и просмотр абонентов.
- Создание абонента и учетной записи вручную.
- Подключение известного email к абоненту текущего хозяйства.
- Создание пользователя и временного пароля как побочное действие подключения, если такого пользователя еще нет.
- Выдача ролей `admin` и `operator` внутри хозяйства.
- Привязка абонента к участкам после проверки документов.
- Просмотр участков.
- Просмотр всего, что видит абонент.
- Просмотр текущего statement по участку и фиксация snapshot-квитанции.
- Просмотр печатной HTML-формы сохраненной квитанции.
- Управление платежными реквизитами хозяйства для печати в новых квитанциях.
- Массовое формирование квитанций по проведенному расчету.
- Постановка сохраненной квитанции в очередь email-доставки.
- Отправка сохраненной PDF-квитанции через SMTP-обработчик очереди.
- Внесение показаний за абонента.
- Исправление ошибочных показаний.
- Внесение оплат вручную.
- Отмена и замена ошибочно внесенной оплаты с причиной.
- Просмотр участков без актуальных показаний перед формированием квитанций.
- Просмотр истории действий.

### Расчет

- Хранение истории тарифов.
- Хранение правил социальных норм по месяцам.
- Расчет потребления по показаниям.
- Расчет начислений по тарифам и нормам.
- Округление денежных компонентов начисления до копеек.
- Расчет знакового баланса как `оплаты - начисления`: положительное значение означает переплату, отрицательное - задолженность.
- Пересчет начислений при изменении тарифов, норм или показаний.

## Не Входит В MVP

- Автоматический доступ к корпоративной почте.
- Автоматический парсинг писем с показаниями.
- Автоматический разбор банковских или бухгалтерских сводок.
- Полностью автоматический запуск ежемесячного расчета и рассылки без действия оператора.
- Интеграция с банком.
- Оплата банковской картой внутри системы.
- Автоматическое оценочное начисление по среднему потреблению без ручного подтверждения.
- Расчет членских взносов.
- Расчет платы за воду.
- Вход через VK, Яндекс, Госуслуги или другие внешние провайдеры.
- Свободная публичная регистрация.

## Требования К Качеству

- Исправления финансово значимых данных должны оставлять след в аудите.
- Расчет должен быть воспроизводимым.
- Исторические тарифы и нормы нельзя просто перетирать без возможности понять, что изменилось.
- Интерфейс администратора должен учитывать работу с абонентами без интернета.
- Система не должна просить абонента повторно оплатить уже закрытые начисления.
- Архитектура аутентификации должна позволять optional TOTP для администраторов.
